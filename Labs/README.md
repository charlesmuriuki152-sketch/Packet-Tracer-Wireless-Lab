# Cisco Packet Tracer Labs

This folder contains comprehensive hands-on labs completed using Cisco Packet Tracer. Each lab demonstrates practical networking and wireless security concepts.

---

## 📚 Lab Overview

### Lab 1: Packet Tracer Tutored Activity - Logical and Physical Mode Exploration
**File:** `1.1.6-packet-tracer-tutored-activitys-logical-and-physical-mode-exploration.pksz`

#### Objectives:
- Understand the difference between Logical and Physical modes in Packet Tracer
- Explore network topology visualization
- Learn how to navigate the Packet Tracer interface effectively

#### Key Concepts Covered:
- **Logical Mode**: View network devices and their connections
- **Physical Mode**: See the physical layout and geographical placement of devices
- Device management and topology design
- Interface switching and mode navigation

#### What I Learned:
- How to visualize network designs in different perspectives
- Importance of understanding both logical and physical network layouts
- Device interconnection principles

#### How to Use This File:
1. Open Cisco Packet Tracer
2. Load this `.pksz` file
3. Switch between Logical and Physical views (tabs at bottom)
4. Examine the network topology

### Lab 2: Network Security Lab 1
**File:** `NetworkSecurityLab1.pkt`

#### Objectives:
- Configure basic network security measures
- Implement secure device configurations
- Understand fundamental network security principles
- Practice routing and switching in a secure environment

#### Network Components:
- Multiple network devices (routers, switches)
- Security-focused configurations
- Network segmentation concepts
- Access control implementation

#### Key Concepts Covered:
- Network device hardening
- Access Control Lists (ACLs) - *if implemented*
- Secure routing protocols
- VLAN configuration for network isolation
- Basic firewall concepts

#### Configurations Practiced:
- Device hostname and password setup
- Interface configuration
- Routing protocol setup
- Network addressing schemes

#### What I Learned:
- How security integrates with network design
- The importance of device and interface configuration
- Basic principles of network segmentation
- How ACLs can control network traffic

#### Challenges & Solutions:
- *Challenge*: Understanding routing in a security context
- *Solution*: Practiced incrementally adding routes while maintaining security

#### How to Use This File:
1. Open Cisco Packet Tracer
2. Load `NetworkSecurityLab1.pkt`
3. Examine the network topology
4. Review device configurations (CLI or GUI)
5. Test connectivity using ping and tracert
